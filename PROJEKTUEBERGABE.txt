STICKY HEADER-ARCHITEKTUR
================================================================
Alle sticky Elemente sitzen im gleichen Scroll-Container (Window).

  1. Haupt-Sticky-Wrapper (ref={headerRef}, position:sticky, top:0, zIndex:10)
       ├── .topbar  ("Vokabel-Trainer", zentriert)
       ├── .tabs    (Listen / Quiz / Statistik / Einstellungen)
       │
       ├── [Quiz-Tab] Zeile 1: liste-detail-header (position:relative)
       │        → Listenname / "X Listen ausgewählt" / "Listen auswählen"
       │        → "(X V.)" Vokabelanzahl grau — nur wenn Liste(n) gewählt
       │        → Toggle-Button (grüner Dreieck) für Listenauswahl
       │
       ├── [Quiz-Tab] Zeile 2: Alle / Bereich (nur wenn Liste gewählt)
       │        → [Alle][Bereich] Toggle
       │        → Bereich aktiv: "(X V.)" gefilterte Anzahl + Expand-Button
       │
       ├── [Quiz-Tab] Sub-Zeile 1: Von–Bis Eingabe
       │        → nur bei Bereich + Expand offen
       │        → leicht eingerückt: Label "Von–Bis" + zwei Zahlenfelder
       │
       └── [Quiz-Tab] Sub-Zeile 2: Liste anzeigen
                → nur bei Bereich + Expand offen
                → "Liste anzeigen / einklappen" + "Auswahl löschen (N)"

  2. headerH = Höhe des gesamten Haupt-Wrappers (per ResizeObserver gemessen)
     → Enthält ALLE Quiz-Zeilen; headerH wächst automatisch mit
     → Quiz-starten-Button und alle anderen sticky Elemente nutzen top:headerH

  3. Sub-Header anderer Tabs (position:sticky, top:headerH):
       - Listen-Tab: .liste-detail-header
       - Statistik-Tab: .statistik-listen-header (ref={statistikListenHeaderRef})

  4. Statistik Ø Score (position:sticky, top: headerH + statistikListenHeaderH)
     → Klebt eine Stufe tiefer als der Listen-Header
     → Enthält "↑ Nach oben"-Button

  5. Quiz "Quiz starten"-Button (position:sticky, top:headerH, zIndex:8)
     → Im Content-Flow nach der Listenauswahl, vor der Config

TRICHTER-LOGIK: VOKABEL-ANZAHL IM QUIZ-TAB
================================================================
Zeile 1  (X V.)  = quizBasisVoks.length   (alle Voks nach Spalten-Filter)
Zeile 2  (X V.)  = quizGefilterteVoks.length  (nach Von-Bis / Checkbox)
Quiz-Button      = verfuegbar  (nach Schlechteste / Unbeantwortet-Filter)
→ Jede Stufe zeigt das Ergebnis der bisherigen Verfeinerung.

Hoisted Berechnungen (vor return(), für Header UND Tab-Content):
  quizKombiListe       = kombinierte Liste aus quizTabListen
  quizBasisVoks        = Vokabeln nach Spalten- / Modus-Filter
  quizGefilterteVoks   = nach Von-Bis / Checkbox gefiltert
  (im Quiz-IIFE als Aliases: kombiListe, basisVoks, gefilterteVoks)


================================================================
SVG ICON-BIBLIOTHEK
================================================================
Alle Icons als Arrow-Functions vor VokabelApp. Prop: s = Größe px.
Farbe: "currentColor" → per CSS oder Elternfarbe steuerbar.

  IcoUp     ({s=12})  Dreieck ▲ (zugeklappt)
  IcoDown   ({s=12})  Dreieck ▼ (aufgeklappt)
  IcoBack   ({s=20})  Zurück-Pfeil (massiv)
  IcoX      ({s=14})  Schließen / Löschen (×)
  IcoPencil ({s=15})  Stift (Spitze unten-links, oben rechtwinklig)
  IcoSpk    ({s=18})  Lautsprecher ohne Schallwellen
  IcoSpkOn  ({s=20})  Lautsprecher mit Schallwellen
  IcoPlus   ({s=15})  Plus-Zeichen (+)
  IcoShare  ({s=15})  Teilen (Pfeil nach oben)
  IcoCopy   ({s=15})  Kopieren (zwei überlagerte Rechtecke)

PFEIL-KONVENTION (konsistent in der gesamten App):
  ▲ (IcoUp)   = eingeklappt
  ▼ (IcoDown) = ausgeklappt

AKTUELLER FEATURE-STAND (nach Session 10)
================================================================

QUIZ-TAB
--------
Sticky Header-Zeilen (alle innerhalb headerRef):

Zeile 1: Listenname + "(X V.)" + Collapse-Button
  - grauer V.-Zähler nur wenn Liste(n) ausgewählt

"Bitte Liste auswählen": großer dezenter Platzhalter-Text
  - erscheint wenn keine Liste gewählt UND Liste eingeklappt

Zeile 2: Alle / Bereich (nur wenn Liste gewählt)
  - Alle: nichts rechts (Anzahl steht in Zeile 1)
  - Bereich: "(X V.)" gefiltert + Expand-Button (btn-toggle-ghost)
  - Klick auf Alle: resettet Checkbox-Auswahl + schließt Liste

Sub-Zeile 1 (nur Bereich + Expand offen): Von–Bis
  - "Von–Bis" Label + Von-Input + "–" + Bis-Input inline
  - leicht eingerückt (28px links)

Sub-Zeile 2 (nur Bereich + Expand offen): Liste anzeigen
  - "Liste anzeigen / einklappen" Button
  - "Auswahl löschen (N)" nur wenn Checkboxen aktiv
  - Einklappen des Expand-Buttons schließt auch die Liste

Info-Toggle Abfrage-Modus: btn-toggle-ghost (▼ öffnet Info-Text)
Quiz-starten-Button: sticky-on-scroll bei top:headerH

States:
  quizBereichEingabeAufgeklappt = Sub-Zeilen Toggle (Bereich-Expand)
  quizTabListen, quizBereichTyp, quizBereichVon, quizBereichBis (wie zuvor)
  quizListeAufgeklappt: für Einzelauswahl-Liste
  quizModusInfoAufgeklappt: Info-Text unter Abfrage-Modus

TASTATUR-QUIZ
-------------
- 3 Fehlversuche erlaubt bevor Score-Abzug + automatische Lösungsanzeige
- 1.+2. Fehlversuch: Box leuchtet 1,5s rot
- 3. Fehlversuch: Score -1, Lösung automatisch angezeigt

DIKTAT-QUIZ
-----------
- Lautsprecher-Buttons: IcoSpkOn (groß, s=52) + IcoSpk (klein, s=16)
- Stimmenfilter: Novelty/Effekt-Stimmen ausgefiltert
- Ergebnisliste: Toggle-Button (Dreieck) statt Text

STATISTIK-TAB
-------------
- Sticky Listen-Header (top:headerH)
- Ø Score sticky bei top:headerH+statistikListenHeaderH (eine Stufe tiefer)
- "↑ Nach oben"-Button im Ø-Score-Header
- Sortierung: 4 Buttons (Score, Streak, Datum, A→Z) mit Richtungs-Toggle
  → State: statistikSort als kombinierter String "score-asc" etc.

LISTEN-TAB
----------
Header Zeile 1 (liste-detail-header):
  - Listenname + Aktionszeile-Toggle (▲/▼) + Stift-Button (btn-ghost-filled)

Aktionszeile (nur wenn aufgeklappt, da darunter):
  Reihenfolge: [+] [Kopieren] [TXT] [Teilen/JSON] [JSON] ... [×]
  - + öffnet Import in bestehende Liste
  - Kopieren: Text in Clipboard
  - TXT: Download als .txt
  - Teilen (IcoShare): teilt JSON-Datei via navigator.share; Fallback Download
  - JSON: öffnet JSON-Export-Modal (mit Optionen)
  - × Löschen (rechts, btn-danger)

Spalten-Stift-Buttons: btn-ghost-filled (heller Hintergrund #f7f5f0)
Vokabeln-Sektion-Header:
  - "Zusammenführen" + "+": background:#fff (auf beigem Hintergrund)
  - Toggle-Button (flat, solid grün) zum Ein-/Ausklappen

Header "Meine Listen":
  - "Neu" Button (IcoPlus) öffnet Import für neue Liste
  - "Exportieren" Button für Multi-Listen-Export

SCROLL-VERHALTEN
----------------
- Quiz-Listen-Toggle ohne Scroll-Sprung:
  listenContainerRef misst Container; toggleListenAuswahl() korrigiert
  window.scrollY via scrollBy()
- Statistik Nach-oben: window.scrollTo({top:0, behavior:'smooth'})


================================================================
SCORING-LOGIK
================================================================
Tastatur/MC/Karte:
  Richtig:    +1, Streak +1 (ab Streak 3: Score min. 2)
  Falsch:     Streak 0, Score -1/-2/-3 je sessionFalschAnzahl
              Erst beim 3. Fehlversuch: Score -1 + Lösung anzeigen
  Aufdecken:  Streak 0, aufgedecktStreak +1

Diktat:
  Richtig:    +1
  Aufdecken:  -1


================================================================
DATENMODELL (localStorage)
================================================================
  vokabel_listen_index     → Array [{id, name}]
  vokabel_liste_<id>       → { id, name, spalten, naechste_id, vokabeln }
      spalten: { E1, E2, D1, D2, i1, i2 }
      vokabeln: [{ id, E1:{wert,falsch:[]}, D1:{wert,falsch:[]},
                   fortschritt:{score,streak,aufgedecktStreak,letzteAbfrage},
                   diktatFortschritt:{score,...} }]
  vokabel_einstellungen    → { modus, autoplay, vorlesen:[] }
  vokabel_session_slots    → Array[6] mit gespeicherten Quiz-Konfigurationen
