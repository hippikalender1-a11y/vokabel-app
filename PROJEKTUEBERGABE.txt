STICKY HEADER-ARCHITEKTUR
================================================================
Alle sticky Elemente sitzen im gleichen Scroll-Container (Window).

  1. Haupt-Sticky-Wrapper (ref={headerRef}, position:sticky, top:0, zIndex:10)
       ├── .topbar  ("Vokabel-Trainer", zentriert)
       ├── .tabs    (Listen / Quiz / Statistik / Einstellungen)
       └── [nur Quiz-Tab] .liste-detail-header (position:relative)
               → Listenname (1 Liste) / "X Listen ausgewählt" / "Listen auswählen"
               → Toggle-Button (grünes Dreieck) für Listenauswahl

  2. headerH = Höhe des Haupt-Wrappers (per ResizeObserver gemessen)
     → Ändert sich je nach Tab

  3. Sub-Header der anderen Tabs (position:sticky, top:headerH):
       - Listen-Tab: .liste-detail-header
       - Statistik-Tab: .statistik-listen-header (ref={statistikListenHeaderRef})

  4. Statistik Ø Score (position:sticky, top: headerH + statistikListenHeaderH)
     → Klebt eine Stufe tiefer als der Listen-Header
     → Enthält "↑ Nach oben"-Button

  5. Quiz "Quiz starten"-Button (position:sticky, top:headerH, zIndex:8)
     → Im Content-Flow nach der Listenauswahl, vor den Slots
     → Scrollt mit hoch bis er am Header anstößt


================================================================
SVG ICON-BIBLIOTHEK
================================================================
Alle Icons sind als Arrow-Functions vor der VokabelApp-Komponente definiert.
Prop: s = Größe in px (Default unterschiedlich).
Farbe: immer "currentColor" → per CSS oder Elternfarbe steuerbar.

  IcoUp     ({s=12})  Dreieck ▲ (zugeklappt)
  IcoDown   ({s=12})  Dreieck ▼ (aufgeklappt)
  IcoBack   ({s=20})  Zurück-Pfeil (massiv, kurzer Stiel, dicke Spitze)
  IcoX      ({s=14})  Schließen / Löschen (×)
  IcoPencil ({s=15})  Stift (Spitze unten-links, oben rechtwinklig)
  IcoSpk    ({s=18})  Lautsprecher ohne Schallwellen
  IcoSpkOn  ({s=20})  Lautsprecher mit Schallwellen
  IcoPlus   ({s=15})  Plus-Zeichen (+)
  IcoShare  ({s=15})  Teilen (Pfeil nach oben)
  IcoCopy   ({s=15})  Kopieren (zwei überlagerte Rechtecke)

VERWENDUNG:
  Toggle-Buttons:  <button className="btn-toggle"><IcoDown/></button>
  Ghost-Buttons:   <button className="btn btn-ghost btn-sm"><IcoPencil/></button>
  Danger-Buttons:  <button className="btn btn-danger btn-sm"><IcoX/></button>
  Topbar-Back:     <button className="topbar-back"><IcoBack/></button>
  Diktat groß:     <IcoSpkOn s={52}/>
  Diktat klein:    <IcoSpk s={16}/>

AKTUELLER FEATURE-STAND (nach Session 9)
================================================================

QUIZ-TAB
--------
- Sub-Header Text: "Listenname" (1 Liste) / "X Listen ausgewählt" (mehrere) /
  "Listen auswählen" (keine) — kompakter als früher
- Listenauswahl: standardmäßig aufgeklappt wenn keine Liste selektiert;
  klappt automatisch auf wenn alle Listen abgewählt werden (useEffect)
- Toggle-Button (grünes Dreieck ▲/▼) statt Text-Buttons
- "Quiz starten (X Vokabeln)"-Button: sticky-on-scroll im Content-Flow
  (nach Listenliste, vor Slots) – klebt bei top:headerH

TASTATUR-QUIZ
-------------
- 3 Fehlversuche erlaubt bevor Score-Abzug + automatische Lösungsanzeige
- 1.+2. Fehlversuch: Box leuchtet 1,5s rot, "Nicht richtig" zentriert
- 3. Fehlversuch: Score -1, Lösung automatisch angezeigt
- State: quiz.aktuelleFehlversuche, quiz.falschFlash

DIKTAT-QUIZ
-----------
- Lösung zeigt: **fettes Wort** – Übersetzung
- Stimmenfilter: Novelty/Effekt-Stimmen ausgefiltert
- Button "Andere Stimme" mit IcoSpkOn-Icon
- Lautsprecher-Buttons: IcoSpkOn (groß, s=52) + IcoSpk (klein, s=16)
- Ergebnisliste: Toggle-Button (Dreieck) statt Text

STATISTIK-TAB
-------------
- Sticky Listen-Header (top:headerH): Listenauswahl mit Toggle-Button
- statistikListenHeaderRef: misst Höhe des Listen-Headers per ResizeObserver
- Ø Score: sticky-on-scroll bei top:headerH+statistikListenHeaderH
  → klebt UNTER dem Listenauswahl-Header (eine Stufe tiefer)
  → enthält rechts "↑ Nach oben"-Button (window.scrollTo top:0)
- Graph: Quiz-Score (grün) + Diktat-Score (orange, gestrichelt)
- Graph-Toggle: "Inkl. unbeantwortete" / "Nur beantwortete"
- Sortierung: 4 Buttons (Score, Streak, Datum, A→Z) mit Richtungs-Toggle
  → Aktiver Button zeigt Pfeil ↑/↓
  → Klick auf aktiven Button dreht Richtung um
  → Default-Richtungen: Score→asc, Streak→desc, Datum→desc, Alpha→asc
  → State: statistikSort als kombinierter String "score-asc" etc.
  → Gilt in beiden Statistik-Views (Tab + Einzelliste)

LISTEN-TAB
----------
- Header-Button "Neu" (+) öffnet Import für neue Liste (früher: "Importieren")
- FAB-Button (kreisförmiges + unten rechts) entfernt
- In einer Liste: Aktionszeile unter dem Header mit:
    [+] [Teilen] [Kopieren] [TXT] [JSON]   [×]
    Plus, Share, Copy per Icon; TXT/JSON als Text; Löschen als IcoX rechts
- Header Zeile 1: Listenname + IcoPencil-Button (öffnet Umbenennen-Modal)
- Header Zeile 2 (Aktionszeile im Content): Alle Aktionsbuttons
- Vokabeln-Toggle: grüner Dreieck-Button statt ▲/▼-Text
- Vokabeln-Import-Button: IcoPlus-Icon (neben "Zusammenführen")
- ✏️-Emojis überall ersetzt durch IcoPencil SVG

SCROLL-VERHALTEN
----------------
- Quiz-Listen-Toggle ohne Scroll-Sprung:
  listenContainerRef misst Container; toggleListenAuswahl() korrigiert
  window.scrollY via scrollBy() wenn Container über Viewport
- Statistik Nach-oben: window.scrollTo({top:0, behavior:'smooth'})


================================================================
SCORING-LOGIK
================================================================
Tastatur/MC/Karte:
  Richtig:    +1, Streak +1 (ab Streak 3: Score min. 2)
  Falsch:     Streak 0, Score -1/-2/-3 je sessionFalschAnzahl
              Erst beim 3. Fehlversuch: Score -1 + Lösung anzeigen
  Aufdecken:  Streak 0, aufgedecktStreak +1 (schwer: -1; nach 3x: Score=-100)

Diktat:
  Richtig:    +1
  Aufdecken:  -1

State-Felder im quiz-Objekt:
  sessionFalsch: {vokId: count}   — Fehlanzahl je Vokabel in dieser Session
  aktuelleFehlversuche: 0-2       — Fehlversuche für die aktuelle Frage
  falschFlash: bool               — true während des 1,5s-Rotblitzes


================================================================
DATENMODELL (localStorage)
================================================================
  vokabel_listen_index     → Array [{id, name}]
  vokabel_liste_<id>       → { id, name, spalten, naechste_id, vokabeln }
      spalten: { E1, E2, D1, D2, i1, i2 }
      vokabeln: [{ id, E1:{wert,falsch:[]}, D1:{wert,falsch:[]},
                   fortschritt:{score,streak,aufgedecktStreak,letzteAbfrage},
                   diktatFortschritt:{score,...} }]
  vokabel_einstellungen    → { modus, autoplay, vorlesen:[] }
  vokabel_session_slots    → Array[6] mit gespeicherten Quiz-Konfigurationen
      Slot-Felder: quizAusgewaehlt, quizFrageTyp, quizAntwortTypenGeordnet,
                   quizInfoTypenSession, quizSpalteModus, quizZeigeInfo,
                   quizModus, quizBereichTyp, quizBereichVon, quizBereichBis,
                   quizReihenfolge, quizSchlechtesteAnzahl, quizSchlechtesteMaxScore,
                   quizUnbeantwortetZuerst, quizDiktatSpalte, quizDiktatUebersetzung
