STICKY HEADER-ARCHITEKTUR (Quiz-Tab)
================================================================
Alle sticky Elemente im gleichen Scroll-Container (Window).

  1. Haupt-Sticky-Wrapper (ref={headerRef}, position:sticky, top:0, zIndex:10)
       ├── .topbar  ("Vokabel-Trainer", zentriert)
       ├── .tabs    (Listen / Quiz / Statistik / Einstellungen)
       └── [Quiz-Tab] Zeile 1: liste-detail-header
                → Listenname / "X Listen ausgewählt" / "Listen auswählen"
                → "(X V.)" Vokabelanzahl grau — nur wenn Liste(n) gewählt
                → Toggle-Button (▲/▼) für Listenauswahl

  2. headerH = Höhe des Haupt-Wrappers (ResizeObserver auf headerRef)

  3. [Quiz-Tab] Alle/Bereich-Zeile (ref={alleBereichRef})
       position:sticky, top:headerH, zIndex:8
       → Eigenständiges sticky Element IM Scroll-Content (nicht in headerRef)
       → [Alle][Bereich] Toggle-Buttons
       → Bei Bereich + Liste offen: [X] Auswahl löschen, [Von–Bis], "(X V.)", [▲/▼]
       → Bei Bereich + Liste zu:    nur "(X V.)" + [▲/▼]
       → Erscheint direkt nach der Listenauswahl (wenn Liste gewählt)

  4. alleBereichH = Höhe der Alle/Bereich-Zeile (ResizeObserver auf alleBereichRef)
       → Dependency: quizTabListen.length

  5. [Quiz-Tab] Quiz-starten-Button
       position:sticky, top:headerH + alleBereichH, zIndex:7
       → Klebt direkt unter Alle/Bereich-Zeile

  6. Sub-Header anderer Tabs (position:sticky, top:headerH):
       - Listen-Tab: .liste-detail-header
       - Statistik-Tab: .statistik-listen-header (ref={statistikListenHeaderRef})

  7. Statistik Ø Score (position:sticky, top: headerH + statistikListenHeaderH)


================================================================
KONTEXT-AUFKLAPP-SYSTEM (Quiz-Tab)
================================================================
Beide Bereiche folgen dem gleichen Prinzip:
  - Aufklappen: nur per Button-Klick
  - Zuklappen: per Button ODER automatisch durch Scrollen
  - Auto-Close: wenn die Unterkante des Inhalts die Oberkante
    des nächsten sticky Elements passiert (Scroll-Listener)
  - Wiederöffnen: scrollt automatisch zum Anfang des Kontexts

LISTENAUSWAHL (quizTabListen auswählen):
  Header:     Zeile 1 (immer sichtbar, in headerRef)
  Inhalt:     ref={listenContainerRef} — Liste verfügbarer Vokabel-Listen
  Auto-Close: listenContainerRef.bottom <= headerH
  Toggle-Fn:  toggleListenAuswahl()
              → beim Öffnen: scrollTo(listenContainerRef.top = headerH)
  Default:    offen wenn keine Liste gewählt (useEffect auf quizTabListen.length)
  CSS-Fix:    overflow:'hidden' + paddingTop:16 (wenn offen) verhindert
              margin-collapse und eliminiert Lücke zum Header

EINZELAUSWAHL / CHECKBOX-LISTE (Bereich-Modus):
  Header:     Alle/Bereich-Zeile (sticky bei top:headerH)
  Inhalt:     ref={einzelauswahlRef} — Checkbox-Liste der Vokabeln
  Auto-Close: einzelauswahlRef.bottom <= headerH + alleBereichH
  Toggle-Fn:  toggleEinzelauswahl()
              → beim Öffnen: scrollTo(einzelauswahlRef.top = headerH + alleBereichH)
  Default:    öffnet automatisch wenn "Bereich" erstmals gewählt
              UND noch keine Auswahl vorhanden (quizCheckboxAuswahl.size === 0)

QUIZ-SEKTION LAYOUT:
  <div className="sektion" style={{paddingTop:0}}>
  → kein Top-Padding damit Alle/Bereich-Zeile direkt am Header klebt
  → Abstand über Listenauswahl via paddingTop auf listenContainerRef

VOKABELANZAHL-TRICHTER:
  Zeile 1  (X V.)  = quizBasisVoks.length   (nach Spalten-/Modus-Filter)
  Alle/Bereich     = quizGefilterteVoks.length  (nach Checkbox-Auswahl)
  Quiz-Button      = verfuegbar  (nach Schlechteste/Unbeantwortet-Filter)

Hoisted Berechnungen (vor return(), außerhalb IIFE):
  quizKombiListe     = kombinierte Liste aus quizTabListen
  quizBasisVoks      = Vokabeln nach Spalten-/Modus-Filter
  quizGefilterteVoks = quizBasisVoks, gefiltert nach quizCheckboxAuswahl
                       (nur wenn quizBereichTyp="bereich" && Auswahl vorhanden)
  (im Quiz-IIFE als Aliases: kombiListe, basisVoks, gefilterteVoks)

REFS FÜR SCROLL-LOGIK:
  headerRef          → ResizeObserver → headerH
  alleBereichRef     → ResizeObserver → alleBereichH
  listenContainerRef → Scroll-Listener (auto-close Listenauswahl)
  einzelauswahlRef   → Scroll-Listener (auto-close Einzelauswahl)
  statistikListenHeaderRef → ResizeObserver → statistikListenHeaderH


================================================================
VON-BIS BEREICHSAUSWAHL (Einzelauswahl / Bereich-Modus)
================================================================
Ermöglicht Auswahl eines zusammenhängenden Vokabel-Bereichs per 2 Klicks.

STATES:
  quizVonBisModus   (bool)      — Modus aktiv
  quizVonBisErster  (null|id)   — erste gewählte Vokabel (Von)

BUTTON-ZUSTÄNDE (sichtbar nur wenn Einzelauswahl aufgeklappt):
  "Von–Bis"  → Modus inaktiv (Default)
  "Von…"     → Modus aktiv, warte auf ersten Klick
  "…Bis"     → Von gesetzt, warte auf zweiten Klick
  Klick auf "Von…" oder "…Bis" bricht Modus ab

ABLAUF:
  1. "Von–Bis"-Button → Modus aktiv ("Von…")
  2. Vokabel klicken  → Von gesetzt ("…Bis"), Zeile gelb + "→" markiert
  3. Vokabel klicken  → Bereich selektiert, Modus beendet ("Von–Bis")

VISUELLE RÜCKMELDUNG:
  "Von"-Zeile: background:#fff3cd, Checkbox orange mit "→"

AUTO-RESET:
  useEffect: wenn quizListeAufgeklappt=false → Von-Bis komplett zurücksetzen

X-BUTTON (Auswahl löschen):
  Sichtbar nur wenn: quizListeAufgeklappt && quizCheckboxAuswahl.size > 0
  → setzt quizCheckboxAuswahl, quizVonBisModus, quizVonBisErster zurück

BEREICH-STATE PERSISTENZ:
  "Alle" klicken: quizCheckboxAuswahl wird NICHT gelöscht (bleibt gespeichert)
  "Bereich" klicken: öffnet Liste nur automatisch wenn quizCheckboxAuswahl.size === 0

SVG ICON-BIBLIOTHEK
================================================================
Alle Icons als Arrow-Functions vor VokabelApp. Prop: s = Größe px.
Farbe: "currentColor" → per CSS oder Elternfarbe steuerbar.

  IcoUp     ({s=12})  Dreieck ▲ (zugeklappt)
  IcoDown   ({s=12})  Dreieck ▼ (aufgeklappt)
  IcoBack   ({s=20})  Zurück-Pfeil (massiv)
  IcoX      ({s=14})  Schließen / Löschen (×)
  IcoPencil ({s=15})  Stift (Spitze unten-links, oben rechtwinklig)
  IcoSpk    ({s=18})  Lautsprecher ohne Schallwellen
  IcoSpkOn  ({s=20})  Lautsprecher mit Schallwellen
  IcoPlus   ({s=15})  Plus-Zeichen (+)
  IcoShare  ({s=15})  Teilen (Pfeil nach oben)
  IcoCopy   ({s=15})  Kopieren (zwei überlagerte Rechtecke)

PFEIL-KONVENTION (konsistent in der gesamten App):
  ▲ (IcoUp)   = eingeklappt
  ▼ (IcoDown) = ausgeklappt

QUIZ-TAB
--------
Zeile 1 (in headerRef, immer sticky):
  Listenname + "(X V.)" + ▲/▼-Toggle für Listenauswahl

Listenauswahl (Scroll-Content, ref=listenContainerRef):
  - Checkbox-Liste aller verfügbaren Vokabel-Listen
  - Schließt automatisch wenn Inhalt oben rausscrollt (→ headerH)
  - "Bitte Liste auswählen" Platzhalter wenn keine Liste + eingeklappt

Alle/Bereich-Zeile (sticky bei top:headerH, ref=alleBereichRef):
  - [Alle][Bereich] Toggle
  - Alle: Vokabelanzahl in Zeile 1 sichtbar; Bereich-Checkbox-Auswahl bleibt gespeichert
  - Bereich + Liste ZU:    "(X V.)" + [▲/▼]
  - Bereich + Liste OFFEN: [X] (wenn Auswahl) + [Von–Bis] + "(X V.)" + [▲/▼]
  - "Bereich" öffnet Liste nur wenn noch keine Auswahl vorhanden

Einzelauswahl (Scroll-Content, ref=einzelauswahlRef):
  - Checkbox-Liste der Vokabeln (nur bei Bereich + aufgeklappt)
  - Schließt automatisch wenn Inhalt oben rausscrollt (→ headerH + alleBereichH)
  - Von-Bis-Bereichsauswahl: 2-Klick-Selektion eines Bereichs

Quiz-starten-Button (sticky bei top:headerH + alleBereichH):
  - Zeigt verfügbare Vokabelanzahl
  - Disabled wenn Konfiguration unvollständig

Von-Bis-Bereichsauswahl: siehe eigenen Abschnitt oben

TASTATUR-QUIZ
-------------
- 3 Fehlversuche erlaubt bevor Score-Abzug + automatische Lösungsanzeige
- 1.+2. Fehlversuch: Box leuchtet 1,5s rot
- 3. Fehlversuch: Score -1, Lösung automatisch angezeigt

DIKTAT-QUIZ
-----------
- Lautsprecher-Buttons: IcoSpkOn (groß, s=52) + IcoSpk (klein, s=16)
- Stimmenfilter: Novelty/Effekt-Stimmen ausgefiltert
- Ergebnisliste: Toggle-Button (Dreieck) statt Text

STATISTIK-TAB
-------------
- Sticky Listen-Header (top:headerH)
- Ø Score sticky bei top:headerH+statistikListenHeaderH (eine Stufe tiefer)
- "↑ Nach oben"-Button im Ø-Score-Header
- Sortierung: 4 Buttons (Score, Streak, Datum, A→Z) mit Richtungs-Toggle
  → State: statistikSort als kombinierter String "score-asc" etc.

LISTEN-TAB
----------
Header Zeile 1 (liste-detail-header):
  - Listenname + Aktionszeile-Toggle (▲/▼) + Stift-Button (btn-ghost-filled)

Aktionszeile (nur wenn aufgeklappt):
  Reihenfolge: [+] [Kopieren] [TXT] [Teilen/JSON] [JSON] ... [×]
  - + öffnet Import in bestehende Liste
  - Kopieren: Text in Clipboard
  - TXT: Download als .txt
  - Teilen (IcoShare): teilt JSON-Datei via navigator.share; Fallback Download
  - JSON: öffnet JSON-Export-Modal (mit Optionen)
  - × Löschen (rechts, btn-danger)

Spalten-Stift-Buttons: btn-ghost-filled (heller Hintergrund #f7f5f0)
Vokabeln-Sektion-Header:
  - "Zusammenführen" + "+": background:#fff (auf beigem Hintergrund)
  - Toggle-Button (flat, solid grün) zum Ein-/Ausklappen

Header "Meine Listen":
  - "Neu" Button (IcoPlus) öffnet Import für neue Liste
  - "Exportieren" Button für Multi-Listen-Export


================================================================
SCORING-LOGIK
================================================================
Tastatur/MC/Karte:
  Richtig:    +1, Streak +1 (ab Streak 3: Score min. 2)
  Falsch:     Streak 0, Score -1/-2/-3 je sessionFalschAnzahl
              Erst beim 3. Fehlversuch: Score -1 + Lösung anzeigen
  Aufdecken:  Streak 0, aufgedecktStreak +1

Diktat:
  Richtig:    +1
  Aufdecken:  -1
