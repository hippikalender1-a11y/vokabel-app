KONTEXT-AUFKLAPP-SYSTEM (Quiz-Tab)
================================================================
Alle fünf Kontexte folgen dem gleichen Prinzip:
  - Aufklappen: nur per Button-Klick
  - Zuklappen: per Button ODER automatisch durch Scrollen
  - Auto-Close: wenn die Unterkante des Inhalts die Unterkante
    des zugehörigen sticky Headers passiert (Scroll-Listener)
  - Wiederöffnen: scrollt automatisch zum Anfang des Kontexts
  - paddingBottom '100dvh' auf der Sektion wenn mind. ein Kontext
    offen → garantiert immer ausreichend Scrolltiefe für Auto-Close

LISTENAUSWAHL (quizTabListen auswählen):
  Header:     Zeile 1 (immer sichtbar, in headerRef)
  Inhalt:     ref={listenContainerRef}
              → OBEN: Gespeicherte Konfigurationen (Slots)
              → UNTEN: Checkbox-Liste verfügbarer Vokabel-Listen
  Auto-Close: listenContainerRef.bottom <= headerH
  Toggle-Fn:  toggleListenAuswahl()
  Default:    offen wenn keine Liste gewählt

VOKABEL-AUSWAHL / EINZELAUSWAHL (Bereich-Modus):
  Header:     Vokabel-Auswahl-Zeile (ref={alleBereichRef}, sticky top:headerH)
  Inhalt:     ref={einzelauswahlRef} — Checkbox-Liste der Vokabeln
  Auto-Close: einzelauswahlRef.bottom <= headerH + alleBereichH
  Toggle-Fn:  toggleEinzelauswahl()
  Default:    öffnet automatisch wenn "Bereich" erstmals gewählt
              UND noch keine Auswahl vorhanden

ABFRAGE-MODUS:
  Header:     Abfrage-Modus-Zeile (ref={abfrageModusRef})
  Inhalt:     ref={abfrageModusContainerRef}
              → Modus-Toggle (Frage–Antwort / wechselnd / Diktat)
              → Info-Text (aufklappbar via btn-toggle-ghost)
              → Diktat-Konfig (conditional, nur wenn Diktat)
              → Spalten auswählen (conditional, nicht bei Diktat)
              → Info-Spalten anzeigen (conditional)
  Auto-Close: abfrageModusContainerRef.bottom <= headerH + alleBereichH + abfrageModusH
  Toggle-Fn:  toggleAbfrageModus()

REIHENFOLGE:
  Header:     Reihenfolge-Zeile (ref={reihenfolgeRef})
  Inhalt:     ref={reihenfolgeContainerRef}
              → Toggle (Zufällig / Schlechteste / Listen-Nr.)
              → Schlechteste-Optionen (Anzahl, Max-Score) wenn Schlechteste
              → Unbeantwortete zuerst (wenn nicht Schlechteste)
  Auto-Close: reihenfolgeContainerRef.bottom <= headerH + alleBereichH + abfrageModusH + reihenfolgeH
  Toggle-Fn:  toggleReihenfolge()

EINSTELLUNGEN:
  Header:     Einstellungen-Zeile (ref={einstellungenRef})
  Inhalt:     ref={einstellungenContainerRef}
              → Lautsprache (Auto-Play, Spalten-Vorlesen)
              → Schwierigkeitsgrad (Einfach / Schwer)
              → "Konfiguration speichern" Button (wenn kannStarten)
  Auto-Close: einstellungenContainerRef.bottom <= headerH + alleBereichH
                + abfrageModusH + reihenfolgeH + einstellungenH
  Toggle-Fn:  toggleEinstellungen()

GESPEICHERTE KONFIGURATIONEN:
  Position:   Oben im Listenauswahl-Kontext (sichtbar beim Öffnen der Liste)
  Slots:      1–5 benannt + Slot 6 "Zuletzt verwendet"
  Klick:      ladeKonfigAusSlot() → schließt ALLE Kontexte
              (listenAuswahl, einzelauswahl, abfrageModus, reihenfolge,
               einstellungen) + lädt Konfiguration

QUIZ-SEKTION LAYOUT:
  <div className="sektion" style={{paddingTop:0, paddingBottom: (offen?) '100dvh' : 0}}>
  → paddingTop:0: kein Top-Padding damit Vokabel-Auswahl-Zeile direkt am Header klebt
  → paddingBottom 100dvh: wenn mind. ein Kontext offen (Scroll-Raum für Auto-Close)

VON-BIS BEREICHSAUSWAHL (Einzelauswahl / Bereich-Modus)
================================================================
Ermöglicht Auswahl eines zusammenhängenden Vokabel-Bereichs per 2 Klicks.

STATES:
  quizVonBisModus   (bool)      — Modus aktiv
  quizVonBisErster  (null|id)   — erste gewählte Vokabel (Von)

BUTTON-ZUSTÄNDE (sichtbar nur wenn Einzelauswahl aufgeklappt):
  "Von–Bis"  → Modus inaktiv (Default)
  "Von…"     → Modus aktiv, warte auf ersten Klick
  "…Bis"     → Von gesetzt, warte auf zweiten Klick
  Klick auf "Von…" oder "…Bis" bricht Modus ab

ABLAUF:
  1. "Von–Bis"-Button → Modus aktiv ("Von…")
  2. Vokabel klicken  → Von gesetzt ("…Bis"), Zeile gelb + "→" markiert
  3. Vokabel klicken  → Bereich selektiert, Modus beendet ("Von–Bis")

AUTO-RESET:
  useEffect: wenn quizListeAufgeklappt=false → Von-Bis komplett zurücksetzen

X-BUTTON (Auswahl löschen):
  Sichtbar nur wenn: quizListeAufgeklappt && quizCheckboxAuswahl.size > 0

BEREICH-STATE PERSISTENZ:
  "Alle" klicken: quizCheckboxAuswahl wird NICHT gelöscht
  "Bereich" klicken: öffnet Liste nur automatisch wenn quizCheckboxAuswahl.size === 0

PUNKT 5 — Slot-Session-Modus ✓ FERTIG
----------------------------------------------
SESSION-GRÖSSE:
  - Im Modal "Konfiguration speichern" einstellbar (Eingabefeld "Session-Größe")
  - Gespeichert in konfiguration.sessionGroesse (0 = kein Session-Modus)
  - Beim Laden eines Slots: wenn sessionGroesse > 0 → Session-Modus aktiv

STATES:
  aktiverSlotNr       (null | slot-nr)  — welcher Slot gerade aktiv ist
  quizSessionGroesse  (number, 0=off)   — Vokabeln pro Mini-Session
  sessionResetConfirm (bool)            — Bestätigungs-Toggle für Reset

SLOT-FORTSCHRITT (im Slot-Objekt, nicht in konfiguration):
  { abgefragt: [vokabel-id, ...], durchlaeufe: number }
  abgefragt  = IDs der in diesem Durchlauf schon abgefragten Vokabeln
  durchlaeufe = wie oft alle Vokabeln komplett durchlaufen (inkrementiert bei Reset nach Abschluss)
  Reset via resetSlotFortschritt() → setzt abgefragt=[], durchlaeufe=0 (komplett)

QUIZ-START (starteQuiz + starteDiktat, Session-Modus):
  1. Basis-Voks aus kombiListe (alle + Bereich-Filter)
  2. sessionGesamt = voks.length (vor Session-Filter)
  3. Filter: abgefragte IDs aus getSlotFortschritt() (liest direkt aus LS → immer aktuell)
  4. Reihenfolge anwenden (schlechteste/zufall/listennr) → kein Slice auf quizSchlechtesteAnzahl
