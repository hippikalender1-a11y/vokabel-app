APP-STRUKTUR
------------
4 Tabs mit persistenter Tab-Leiste:

  LISTEN-Tab
    - Listenübersicht (alle Vokabellisten)
    - Listen-Detail (Spalten, Vokabeln ein-/ausklappbar)
    - Importieren (Text einfügen → analysieren → Spalten zuweisen → importieren)
    - KI-Prompt generieren (Thema-basiert oder Foto-Umwandlung)

  QUIZ-Tab
    - Sticky Sub-Header mit Listenauswahl-Info + Ausklappen/Einklappen-Button
    - [GEPLANT] Sticky "Quiz starten"-Button als zweite Header-Zeile
    - Multi-Listen-Auswahl mit Checkboxen (einklappbar)
    - Gespeicherte Konfigurationen (5 Slots + "Zuletzt verwendet")
    - Quiz-Modi: Sequenziell, Rotierend, Diktat
    - Spalten-Konfiguration (Frage / Antwort / Info, je Modus: Tastatur / MC / Karte)
    - Vokabel-Auswahl: Alle, Bereich (Von–Bis), manuelle Checkbox-Auswahl
    - Reihenfolge: Zufällig, Schlechteste N, Listen-Nr.
    - Lautsprache: Auto-Play (nur Englisch), Spalten-Auswahl
    - Schwierigkeits-Modus: Einfach / Schwer

  STATISTIK-Tab
    - Sticky Header mit Listenauswahl (ähnlich wie Quiz-Tab)
    - Sortierung, Filterung, Score-Anzeige

  EINSTELLUNGEN-Tab
    - App-Version und Neu laden
    - Anzahl gespeicherter Listen

STICKY HEADER-ARCHITEKTUR
--------------------------
Alle sticky Elemente sitzen im gleichen Scroll-Container (.app).

  1. Haupt-Sticky-Wrapper (ref={headerRef}, position:sticky, top:0, z-index:10)
       ├── .topbar  ("Vokabel-Trainer", text-align:center)
       ├── .tabs    (Listen / Quiz / Statistik / Einstellungen)
       └── [nur Quiz-Tab] .liste-detail-header (position:relative)
               → "Listen auswählen" (grau) ODER
                 "X Vokabeln aus Y ausgewählten Listen"
               → Button "Ausklappen / Einklappen"

  2. headerH = Höhe des Haupt-Wrappers (per ResizeObserver gemessen)
     → Ändert sich je nach Tab (Quiz-Tab hat zusätzliche Zeile)

  3. Sub-Header der anderen Tabs (position:sticky, top:headerH):
       - Listen-Tab: .liste-detail-header (Meine Listen / Importieren / Exportieren)
       - Statistik-Tab: .statistik-listen-header

  4. quiz-liste-sticky (Vokabellisten-Range-Header, position:sticky, top:headerH)
     → Erscheint nur wenn quizListeAufgeklappt=true (Vokabel-Checkboxen sichtbar)

QUIZ-LOGIK
----------
Scoring (Tippen/MC/Karte):
  Richtig:    +1 Score, Streak +1 (ab Streak 3: Score min. 2)
  Falsch:     Streak Reset, Score -1/-2/-3 je nach Fehlerhäufigkeit
  Aufdecken:  Streak Reset, aufgedecktStreak +1
              (schwer-Modus: -1; nach 3x aufdecken: Score = -100)

Diktat-Scoring:
  Richtig:    +1
  Aufdecken:  -1 (je falschem Versuch ein Buchstabe mehr aufgedeckt)

Multi-Listen-Quiz:
  Vokabeln erhalten kombinierte ID: "<listeId>__<vokId>"
  Scores werden in die Originalliste zurückgeschrieben (_listeId, _origId)


QUIZ-ANSICHTEN (Render-Reihenfolge)
------------------------------------
  1. Diktat – Fertig-Screen  (phase === "fertig", modus === "diktat")
  2. Diktat – Aktiv          (modus === "diktat")
  3. Quiz – Fertig-Screen    (phase === "fertig")
  4. Quiz – Aktiv            (sequenziell / rotierend)
  5. Statistik               (ansicht === "statistik")
  6. Haupt-Layout            (Tabs, Listen, Import, KI-Prompt, Einstellungen, Quiz-Setup)


WICHTIGE VERHALTENSREGELN
--------------------------
- TTS (🔊) NUR für englische Spalten (E1, E2) – niemals Deutsch
- "Bis Nr." im Bereich-Modus muss explizit eingegeben sein,
  sonst ist der Range-Filter inaktiv (nur Checkbox-Auswahl gilt)
- MC-Modus: Nach richtiger Antwort oder "Lösung anzeigen" werden
  die MC-Buttons ausgeblendet; die Antwort erscheint als einzelnes
  Wort in Fragetext-Formatierung (1.5rem, fett) in der Antwortbox
- Tabs sind immer sichtbar (kein eigener Zurück-Button im Quiz)
- Nach Quiz: immer zurück in den Quiz-Tab


IMPORT / EXPORT
---------------
TXT-Export/Import:
  - Spaltentrennzeichen: //
  - Falsche Antworten (MC): || gefolgt von | als Einzeltrenner
  - Phonetische Umschriften [...] werden beim Import entfernt
  - Header-Zeile mit Typ-Tags: "Englisch [E1] // Deutsch [D1]"
    → Tags sind maßgeblich für Spaltenzuordnung (nicht Spaltenname)
  - Mehrere Listen in einer Datei: # ListenName als Abschnittstrenner
  - Einzelne Liste: einfach ohne # starten

JSON-Export/Import (empfohlenes Backup-Format):
  - Exportiert: Vokabeln (immer), Falsche Antworten, Quiz-Fortschritt,
    Diktat-Fortschritt (jeweils optional per Checkbox)
  - Import: erkennt automatisch wenn Datei mit [ oder { beginnt
  - Dateiname: mehrere Listen → "vokabel-backup.json",
               eine Liste → "<listenname>.json"
  - Zuverlässiger als TXT für Gerät-zu-Gerät-Transfer

Beide Formate werden direkt heruntergeladen (kein Share-API,
da auf Android die Datei nur 2 Bytes ankam via Mail).


BEKANNTE BESONDERHEITEN
------------------------
- Der //-Trenner im Import trennt Spalten
- ||  leitet falsche MC-Antworten ein, | trennt Einzelantworten
- Phonetische Umschriften [...] werden beim Import automatisch entfernt
- Die Vokabelliste unter "Liste anzeigen" zeigt alle Vokabeln,
  auch wenn noch keine Quiz-Spalten konfiguriert sind
- Quiz-Konfigurationen (Slots) speichern KEIN Checkbox-Zustand

ÄNDERUNG 2 – "Quiz starten"-Button als permanente zweite Header-Zeile
-----------------------------------------------------------------------
Ziel: Der wichtigste Button bleibt immer sichtbar, egal wo man scrollt.

Verhalten:
  - Nur sichtbar wenn mindestens eine Liste ausgewählt ist (quizTabListen.length > 0)
  - Position: direkt unter dem "Listenauswahl"-Header
    → Bei eingeklappter Liste: gleich unter dem Header
    → Bei ausgeklappter Liste: unter den sichtbaren List-Einträgen,
      ABER wenn man runterscrollt und der Button aus dem Viewport geht,
      klebt er als "zweite Header-Zeile" direkt unter dem Haupt-Header
  - Das bedeutet: der Button IST IMMER IM HAUPT-STICKY-WRAPPER enthalten
    (wie der Quiz-Listenauswahl-Header), damit er nie verschwindet
  - Reihenfolge im sticky wrapper:
      Topbar → Tabs → Quiz-Listenauswahl-Header → [Quiz-Starten-Button wenn Liste gewählt]

  Implementierung:
  - Button direkt im headerRef-Div als zusätzliche Zeile einfügen
  - Conditional: {tab === "quiz" && quizTabListen.length > 0 && (...)}
  - Button-Text: "Quiz starten (X Vokabeln)"  ← zeigt gefilterte Anzahl hier!
  - Button klickt starteQuiz() auf
  - disabled wenn !kannStarten (keine Frage/Antwort-Spalte konfiguriert)
  - Unter dem Button im Scroll-Bereich: Gespeicherte Slots kommen zuerst,
    dann die restlichen Optionen (Abfrage-Modus, Spalten etc.)

  WICHTIG: Da headerH per ResizeObserver automatisch gemessen wird,
  passt sich alles andere automatisch an die neue Höhe an.


ÄNDERUNG 3 – Neue Quiz-Option: "Unbeantwortete Vokabeln zuerst"
----------------------------------------------------------------
Neuer State: quizUnbeantwortetZuerst (useState(true))

Verhalten:
  Wenn aktiviert (Standard: an):
  → Im Modus "Zufällig": Erst aus allen Vokabeln ohne Score-Eintrag
    (fortschritt?.score === undefined oder null) zufällig auswählen,
    bevor bereits beantwortete Vokabeln zufällig ausgewählt werden.
  → Im Modus "Listen-Nr.": Vokabeln ohne Score-Eintrag zuerst (in
    Listennummer-Reihenfolge), dann bereits beantwortete.
  → Im Modus "Schlechteste": Keine Auswirkung (dort geht es ja
    explizit um beantwortete Vokabeln mit schlechtem Score).

  Umsetzung in starteQuiz()/starteDiktat():
  - voks = gefilterteVoks.filter(v => !v.fortschritt?.score && score !== 0)
    (oder: score === undefined/null = "nie beantwortet")
  - Bei Sequenziell/Rotierend: fortschritt?.score prüfen
  - Bei Diktat: diktatFortschritt?.score prüfen

  UI-Platzierung:
  - Im Abschnitt "Reihenfolge", unterhalb der Zufällig/Schlechteste/Listen-Nr. Buttons
  - Als Checkbox-Zeile: [✓] Unbeantwortete Vokabeln zuerst


ÄNDERUNG 4 – "Schlechteste"-Modus: Max-Score-Filter
-----------------------------------------------------
Neuer State: quizSchlechtesteMaxScore (useState(""))  ← leer = kein Filter

Verhalten:
  Aktuell: "Schlechteste N" = die N Vokabeln mit dem niedrigsten Score
  Neu: "Schlechteste N mit Score < X"
  → Erst alle Vokabeln filtern, deren Score < X (quizSchlechtesteMaxScore)
  → Von diesen dann die N schlechtesten nehmen
  → Sind weniger als N verfügbar, werden alle gefilterten genommen
  → Sind mehr als N verfügbar, werden die N schlechtesten genommen

  Beispiel: 50 Vokabeln, MaxScore = -5, Anzahl = 20:
    - Filter: alle mit score < -5 → z.B. 30 Vokabeln
    - Davon die 20 schlechtesten → 20 Vokabeln für Quiz

  Beispiel: 50 Vokabeln, MaxScore = -5, Anzahl = 40:
    - Filter: score < -5 → z.B. 30 Vokabeln
    - Davon die 40 schlechtesten → aber nur 30 verfügbar → 30 Vokabeln

  UI-Platzierung:
  - Im "Schlechteste"-Abschnitt unter dem Anzahl-Input
  - Optional: Input "Max. Score (leer = kein Filter)" mit number-Input
  - Label: "Score kleiner als:" [  -5  ]

  Für den Quiz-Header (Vokabel-Anzahl):
  - Bei Schlechteste + MaxScore-Filter: Anzahl der tatsächlich
    gefilterten Vokabeln anzeigen (wie verfuegbar im IIFE berechnet)
  - Die bisherige Gesamt-Anzeige bleibt für alle anderen Modi

  State in speichereKonfigInSlot/ladeKonfigAusSlot:
  - quizSchlechtesteMaxScore muss mit gespeichert/geladen werden
